Hexo的搭建

hexo博客搭建

一.安装Node.js和git

node.js是一款主要用于服务器端开发的工具.

可以做到让Javascript直接运行在服务器上，用于构建高性能的网络应用
主要用途为构建Web服务器.实时应用.API服务.代理服务器,服务器渲染等
安装于D:\New Folder\node.js

Git 是分布式版本控制工具.

常用于代码管理、团队开发、版本发布、文档追踪
​记录变更,保存代码/文件的所有修改历史，可回退到任意版本
​高效协作,多人同时修改同一项目，自动合并或提示冲突
分支管理,独立开发新功能（分支），测试通过后合并到主代码

数据安全,本地和远程仓库双重备份，代码永不丢失
基础命令:commit(提交).push/pull(同步).merge合并).clone(下载仓库)
安装于C:\Program Files\Git
注:git的作用中大部分可被github desktop替代(基于Git的远程代码托管平台)
而Git作为本地版本控制工具，核心是管理文件变更
检验

意为成功安装了 1 个包
当前项目依赖的 59 个包（开源项目）正在寻求资支持可使用npm fund查看相关详情
关闭资助提示(配置 npm 全局忽略资助提示)

二.安装Hexo

Hexo是一个基于Node.js的静态博客生成器.可将 Markdown 文件快速渲染成静态网页.支持主题和插件，适合搭建轻量.高效的个人博客或文档站点.
在全局安装 Hexo 命令行工具

三.Github配置与本地博客搭建

建立Github仓库,在GitHub/GitHub desktop中新建一个库并起名my-blog，拉取或推送上去，保证两端同步
在my-blog中(在终端中cd my-blog/或在my-blog中git bash here)
hexo g hexo generate 的简写
读取 source/_posts 目录下的 Markdown 文章，将其转换为 HTML
将所有生成的 HTML、CSS、JS 文件保存到 public 文件夹

四.设置SSH

SSH(Secure Shell)是一种加密网络协议,用于在不安全的网络>中(如互联网)安全地远程访问和管理计算机系统.传输文件或执>行命令.它的核心目标是替代早期不安全的协议(如Telnet.>FTP).通过加密和身份验证机制保护通信安全.
SSH的主要用途
​1.远程终端访问,登录到服务器执行命令行操作.例如：

2.安全文件传输,通过 scp 或 sftp 加密传输文件：

3.端口转发与隧道,加密其他协议流量(如通过SSH隧道访问受限的数据库或网站)。
​4.自动化脚本与部署,结合密钥认证，实现无人值守的远程操作(如CI/CD流水线)

在git文件夹(C:\Users\26820.git)下

cd ~/.ssh

1.​cd
全称是Change Directory(qie换目录),用于在终端中切当前工作目录.
​2.~
符号~代表当前用户的​主目录​(Home Directory).如：
Linux/Mac：/home/用户名/ 或 /Users/用户名/
Windows（WSL或Git Bash）：C:\Users\用户名
​3..ssh
这是一个隐藏文件夹(以.开头的文件/夹默认隐藏),通常用存放 ​SSH相关配置和密钥，例如:
id_rsa 和 id_rsa.pub(SSH私钥和公钥)
known_hosts(记录信任的远程服务器信息)
config(自定义SSH连接配置)

ssh-keygen -t rsa -C ‘‘2682076246@qq.com’

是一个用于生成SSH密钥对​(公钥和私钥)的命令，常用于安全登录远程服务器(如GitHub.Linux服务器等)
1.​ssh-keygen
SSH 密钥生成工具,用于创建加密的密钥对.
​2.-t rsa
指定密钥类型为RSA算法​(一种非对称加密算法,广泛兼容且安全)
3.-C ‘‘2682076246@qq.com’
远程服务器登录,将公钥添加到服务器的~/.ssh/authorized_keys文件,实现免密登录
去看回显中的路径找到id_rsa.pub，记事本打开并全部复制。

在Github上头像找到Settings->SSH and GPG keys->New SSH key，名字随意
